A time when I got blocked on a simple problem:

What was the problem?

When I was first introduced to HTML and CSS, I had a an issue with my web pages not linking up and the image links always breaking. I was using Adobe Dreamweaver, and it had a feature where you could switch between code and visual views. I would use the visual view to place images and add links to other files in my website, so I didn't fully understand the concept of linking images and other pages properly.

As a result I would move files around in the directory not known I was breaking the links in my website, and every time I would open up Dreamweaver to work on my assignment, all of the links were broken and I was wondering why.

What problem solving techniques I used?

Techniques I used to try and solve this problem included Googling on the internet, asking peers and trying something.

Being new to Web Development I didn't understand the web terminology and found it hard to ask the right questions and describe the issues I was facing, so my searches weren't returning the right results.

When asking my peers, it was abit easier to describe the issues as I was able to show people the problems on my computer. However, I found a large majority were also struggling with a similar issue and were also very new to Web Development.

In the end, playing around with the code and trying different things actual help my solve the probelms I was having with broken links and as a result helped me understand why it was happening and how to solve it in the future.

How did I feel throughout the process?

Initial frustrated, but by the end relieved and quiet proud that I was able to solve the issue myself. I also felt a sense of enlightenment because I felt I was starting to understand the code abit better.

What did I learn?

Learning outcomes from this experience included, taking the time to examine the code and trying to understand what each part is trying to do and determining the folder structure of your website before jumping into the code, so there is less chance of restructing late in the project leading to broken links etc.

How I solved a problem in an elegant way:

What was the problem?

A recent project I worked on was a Wordpress website with a custom built theme developed by an overseas company which had custom intergrations with a recruitment service API.

My initial involvement with this project was to implement frontend changes to improve user experience, but later was asked to investigate an issue with a feature that alerts users when a new job within a department they are interested in is posted.

What problem solving techniques I used?

The first techique I used was inspect the error log in the developer tools. I identified an error message which confirmed the problem was occuring, I was able to trace the error back to the file generating the error. I also used Google to find out more information about the error, the various conditions that could cause this error and suggestions to fix the error.

How did I feel throughout the process?

I felt good throughout the problem solving process, with a number of years experienceas a web developer made it easier tackling this problem. I have become familiar with the various tools available to developers and how to use them, so investigating the issue wasn't as stressful as it would have been if I was new to web development.

What did I learn?

I learn't the importance of the call stack feature of the developer tools. Being able to run your javascript code line by line, adding breaks, seeing what values are being returned, seeing which methods and/or variables are being hoisted and observing the order methods are being executed.

Problem Solving Techniques & Process Reflection:

Pseudocode:

I find pseudocoding a part of my development process, it helps me map out what I need to do before attempting to write any code.

It also ensures I have a clear understanding and helps with finding solutions to problems.

Trying Something New:

Trying things is something I do alot for CSS problems particularly in the developer tools to try and find solutions to style and layout probelms and learn.

Rubber Ducky Method:

Never tried but could see the advantages of voicing the problem.

Reading Error Messages:

Reading error messages is the first place I start whenever I have a problem relating to web development.

Console.loggin:

Console.logging is a great feature of JavaScript to inspect the return values of a variable or method. When I write my JavaScript methods, I always log variables in my development environment whenever I am expecting to alter the values to check the returned values are what was expected.

Googling:

Google is a great asset for any developer, there is a high chance that someone has encountered the same or similar issue and have posted the problem on the internet. 9 out of 10 times, I have found an answer to, or found a suggestion to a problem I have encountered in the past.

Asking Peers For Help:

Asking peers for help is a good habit to build. I find talking to with your peers can not only help you with your problem, but you may be able to help other as well and build relationships.
