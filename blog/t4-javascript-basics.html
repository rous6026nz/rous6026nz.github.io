How does JavaScript compare to HTML and CSS?

JavaScript provides the interactive layer of a website, and is generally used to provide feedback to user interactions and serve up dynamic content. HTML is used to mark-up the structure of a webpage, CSS is the visual layer of a webpage used to style elements and both are static programing languages.

The disadvantage of JavaScript is that it is an imperative language and is fragile, were as HTML and CSS are declarative languages and are resilient. The disadvantage of being a imperative language, is if there a mistake or something unknown in your JavaScript code, even as small as missing a semicolon the application will break.

Where as with declarative languages like HTML and CSS, if the browser comes across some code that it doesn't understand i.e a mistake or code it doesn't support, it will be skipped and the parser will move on.

Explain control flow and loops using an example process from everyday life?

A simple everyday process which I could use to explain control flow and loops would be the process of making breakfast in the morning. In my household we have 5 people who I will need to cook breakfast for, each person requires 2 pieces of toast.

So my control is I need to cook 10 pieces of toast, and my loop is toasting 2 pieces of toast for every iteration. Once I have toasted 10 pieces of bread, my loop ends because I have reached my control.

Explain the difference between accessing data from arrays and objects?

When accessing data in an array, you access the data using bracket notation and reference the position of the data you want to retrieve using indexing.

When accessing data in an object, you access the data using either bracket notation or dot notation and reference the data you want to retrieve using keys.

Explain what functions are and why they are useful?

Functions are blocks of code that perform a specific task. Common uses of functions are to automate time intensive or repeataive tasks through to small tasks like formatting text or processing an array.

The use functions help speed up development and programming DRY (Don't Repeat Yourself) which means we can create the function once and use it in multiple places, and can have arguments and produce differing results when called.
